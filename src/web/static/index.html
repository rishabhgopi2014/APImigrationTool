Deployment Steps

1

Download All YAML Files

Click the download buttons above to save each YAML file to your local machine.

2

Verify Kubernetes Cluster Connection

Ensure you're connected to the correct cluster:

kubectl config current-context kubectl get namespace gloo-system
3

Deploy to Kubernetes

Apply the configurations in this order:

# Apply Upstream (backend service)
kubectl apply -f upstream.yaml
# Apply VirtualService (routing rules)
kubectl apply -f virtualservice.yaml
# Apply Auth Config (if generated)
kubectl apply -f authconfig.yaml
# Apply Rate Limit (if generated)
kubectl apply -f ratelimitconfig.yaml
4

Verify Deployment Status

Check that resources are created successfully:

# Check VirtualService status kubectl get virtualservice -n gloo-system # Check Upstream status kubectl get upstream -n gloo-system # View detailed status kubectl describe virtualservice {{ selectedAPI.name }}-vs -n gloo-system
5

Test Your API

Send a test request through Gloo Gateway:

# Get Gloo proxy URL export GATEWAY_URL=$(kubectl -n gloo-system get svc gateway-proxy -o jsonpath='{.status.loadBalancer.ingress[0].ip}') # Test the API curl -v http://$GATEWAY_URL{{ selectedAPI.basePath }}/test

Important Notes

  • Ensure Gloo Gateway is installed in your cluster before applying these configs
  • Update the upstream backend URL if needed (check upstream.yaml)
  • Test in a development environment first before production deployment
  • Monitor logs for any errors: kubectl logs -n gloo-system deployment/gateway-proxy -f
